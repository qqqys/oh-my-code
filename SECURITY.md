# Security policy

## Supported versions

Security fixes are provided for the latest published release.

## Reporting a vulnerability

Do not open a public Issue for a suspected vulnerability or credential exposure. Use GitHub private vulnerability reporting for this repository and include reproduction steps, affected versions, impact, and any proposed mitigation.

Never include live credentials, private repository content, or unredacted diagnostic bundles in a report.

## Product security boundaries

Oh My Code treats repository scope, tool approval, command policy, secret redaction, durable idempotency, and Git safety as product behavior. A pull request that weakens one of these boundaries must include focused regression coverage and explicit end-to-end evidence.
